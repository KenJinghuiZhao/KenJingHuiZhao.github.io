《KenBrowser Privacy Policy》

Dear KenBrowser users,

Welcome using KenBrowser! This is a document that specify the privacy policy regarding KenBrowser app. KenBrowser is a green and light-weight mobile browser for Android platform. It does not require user account registration, and it does not include functions like message/info pushing, social networking etc, so there is no any data will be uploaded/transfered from the app to anywhere.

Policy defined date：2023-8-18

This policy document covered below several areas:

  • The functions in this app that may involve data saving
  • What kind of data we save and how we use
  • How we protect your privacy
  • How we update this policy document
  • How to contact us

The functions in this app that may involve data saving

As mentioned above, KenBrowser has no any functions like user account registratin, message/news pushing, social networking etc., so we will NOT collect any user personal data like user name, account password, mobile phone number etc. However, to better serve the webpage browsing functions, we do have below modules that will save user operation parameters.

1）Menu - Favorite List
Data saved： The title and url of the favorite websites that user input.
How to save： All favorite bookmarks are saved in a html file that stored in app's private internal storage.
Where to save： app's private internal storage
How we use it： when starting the app, will read all the bookmarks in html file, and show it as the homepage of the app. User can edit/add/delete bookmarks via Menu-FavoriteList menu item.
When will delete： when the app is uninstalled, Android system will automatically delete all the bookmark data that stored.
Will data be uploaded or shared? all the data in html file, will only be stored in user's phone storage, will NEVER be uploaded or shared to any remote server or 3rd parties.

2）Menu - History List
Data saved： when user close a webpage, will add the page title and url into the history list. Will save up to 20 items.
How to save： save in the app's runtime memory as a linked list.
Where to save： in the app's own runtime memory in Android system, no any persistance storage. Will be cleared once the app exit.
How we use it： When user click Menu-History List, will show all the saved history items to let user recover the pages.
When will delete： once the app exit or be closed, the data will be cleared automatically.
Will data be uploaded or shared? data won't be shared with any other apps in device, any won't be uploaded or shared to any servers.

3）Menu - Setup - Injected Javascript
Data saved： when user choose a javascript file and share to KenBrowser, the app will save the file.
How to save： The file is saved in text/plain format with .js file type.
Where to save： File is saved in the app's private internal storage.
How we use it： When user open the target webpage, if the page hit the script fileter, then the javascript will be injected in webpage and display in KenBrowser.
When will delete： When user uninstall the app, all javascript file will be cleared by Android system automatically.
Will data be uploaded or shared? data won't be shared with any other apps in device, any won't be uploaded or shared to any servers.

4）Menu - Check crash log
Data saved： when the app crash unexpectedly, the app will save the device model, app version in a local html file for better debugging.
How to save： all data be saved in a html file name "exception_log.html"
Where to save： the file is saved in app's private internal storage.
How we use it： when the app was crash out, the user can enter the app again, and click menu-CheckCrashLog, then the html file will be showed in a new tab. User can decide whether to send author those information for debugging.
When will delete： when the app is uninstalled, Android system will automatically delete all the data that stored.
Will data be uploaded or shared? data won't be shared with any other apps in device, any won't be uploaded or shared to any servers.

5）The Chrome WebView core that KenBrowser based on
Data saved： KenBrowser is developed based on Chrome WebKit WebView core. So same as all those browsers based on WebKit, cache or cookie may be saved by WebKit core.
How to save： all cache and cookie saving are defined by WebKit core, KenBrowser has no any control on it.
Where to save： data are saved in app's private internal storage.
How we use it： data will be used by Chrome WebKit core to speed up webpage openning speed, KenBrowser has no any access on those data.
When will delete： when the app is uninstalled, Android system will automatically delete all the data that stored.
Will data be uploaded or shared? all there data be saved and accessed by Chrome WebKit core, and will be complied to Android privacy policy. KenBrowser has no any access to those data.


How we protect your personal data.

KenBrowser is an app that installed and run in user mobile device locally. It does not connect to any server, remote database or cloud platform, so we do not share or transfer any user local data/information to anywhere. All the saved data we mentioned above, will only be stored in user local device, and will be cleared once user uninstall the app.


How we update this policy document

While KenBrowser vesion evolve in future, the privacy policy may change accordingly. All changes will be updated in this webpage, and history version will be kept. For major changes, we will also specify in app's homepage to notify users.
